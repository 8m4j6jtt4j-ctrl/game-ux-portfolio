게임플레이 이벤트_보물찾기 UX

이벤트성 신규 게임 콘텐츠가 들어갈 때 인게임/아웃게임 UX를 함께 정리한 사례입니다.

Role
게임플레이 이벤트 UX 스크립트, 인게임/아웃게임 흐름 정리
Period
2025.12 / 2026.02

Planning Context

어떤 기획이었나

  • 게임플레이 이벤트는 기간성 이벤트처럼 보이지만, 실제로는 신규 콘텐츠 규칙과 보상/진행 상태가 함께 들어갑니다.
  • 인게임에서 행동하는 흐름과 아웃게임에서 상태를 확인하는 흐름이 분리되면 유저가 이벤트 규칙을 놓치기 쉽습니다.

Process

어떻게 작업했나

  • 이벤트의 진입 조건, 진행 규칙, 보상 확인, 일일/기간 제한, 배지와 스트링을 나눠 정리했습니다.
  • 인게임 플레이 중 필요한 안내와 아웃게임 UI에서 확인해야 하는 정보의 역할을 분리했습니다.
  • 데이터 테이블과 스크립트 문서가 같은 이벤트 규칙을 가리키도록 확인했습니다.

Deliverables

결과물

  • 3UD_게임플레이 이벤트_보물찾기_스크립트.pptx
  • 보물찾기_Badge.xls
  • 보물찾기_ClientString.xls
  • 보물찾기_GamePlayEvent.xls

Result

도출한 결과

이벤트성 신규 콘텐츠의 인게임/아웃게임 접점을 분리해 유저 안내와 데이터 기준을 함께 정리했습니다.

Deliverable Captures

산출물: UX 스크립트 문서 캡처

원본 문서는 포함하지 않고, 포트폴리오 확인용으로 일부 페이지만 이미지화했습니다. 슬라이드 마스터/상하단 영역은 흐림 처리했습니다.
